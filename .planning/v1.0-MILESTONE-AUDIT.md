# Milestone v1.0 — Audit Report

**Audited:** 2026-05-18
**Status:** gaps_found
**Phases:** 11/11 individually verified (passed) — but 4 cross-phase integration breaks reduce 4 functional/non-functional requirements to partial status

## Summary

All 11 individual phases passed their goal-backward verification (combined score: 11/11 phases, 218/218 plan-level must-haves). However, the milestone-level audit found four cross-phase wiring gaps that reduce four requirements (FR2, FR4, FR5, NFR4) from satisfied to partial.

The breaks fall into a recognizable pattern: **late-phase data shape additions (Phases 9, 10, 11) didn't always update the cross-cutting code paths owned by earlier phases.** The schema-first codegen pipeline established in Phase 02.1 was bypassed for AuditEntry (Phase 10) and ReconciliationOutcome / AuthorityBinding / ResultPatch (Phase 11). The template_service in Phase 1 was extended in Phase 03.1 (prompt_template) but not in Phase 11 (authority_bindings). The CockpitBadge in Phase 9 was not updated when Phase 11 added the reconciliation Link2 icon to Results-view ValidationBadge. The Phase 9 PATCH endpoint stores edited_data but the frontend never reads it back.

None of the breaks corrupt data; they degrade UX or persistence-on-reload behavior. All are local-scope fixes — the cumulative gap-closure phase is probably 5-7 small tasks.

## Cross-Phase Integration Issues

### Critical (block requirements)

1. **template_service silently drops authority_bindings** — Same regression pattern as Phase 03.1's prompt_template fix (which was caught and fixed in 03.1 Plan 03). Phase 11 added the field on the create/update API but didn't extend `create_template` and `update_template` in `apps/backend/app/services/template_service.py`. Save → reload → load template → bindings missing → ReconcilePane shows no bindings → URI emission silently skipped. **Affects FR2 + FR4.**

2. **handleCellReconciled(null) PATCH omits clear_reconciliation** — `outcome ?? undefined` evaluates to `undefined`, which JSON.stringify drops. Backend never receives the clear signal. Zustand is updated correctly so UI looks right, but checkpoint.json retains the stale reconciliation. Curator clears → reloads → reconciliation reappears. One-line fix in `CleanStep.handleCellReconciled`. **Affects FR4 + FR5.**

3. **CockpitBadge in Verify does not render reconciliation icon** — Phase 11 added the Link2 icon + tooltip to `apps/frontend/src/features/results/ValidationBadge.tsx` (Results view) but did not propagate the change to `apps/frontend/src/features/verify/CockpitBadge.tsx`. Curators in Verify cockpit cannot see whether a field has been reconciled — they have to switch to Results to check. **Affects FR4 UX consistency.**

4. **edited_data PATCH writes never read back into ExtractionResult type** — Phase 9 PATCH endpoint stores curator edits to checkpoint.json. `GET /results` returns it as part of the raw response. But `ExtractionResult` (TS + Pydantic) has no `edited_data` field, so the frontend hydration logic in `ResultsStep` and `VerifyStep` only reads from Zustand persist. localStorage clear / browser change / different device → edits stranded in backend but lost from UI. **Affects FR5.**

### Non-critical (tech debt)

5. **Schema codegen pipeline bypassed for 4 phases** — `packages/shared-types/generated/` is never imported. `AuditEntry` (Phase 10), `ResultPatch` (Phase 9), `AuthorityBinding` + `ReconciliationOutcome` (Phase 11) never added to JSON Schema. Parallel hand-maintained types in `batchesApi.ts` and `schemas.py` are the de-facto source of truth. **Affects NFR4.**

6. **POST /revalidate endpoint orphaned** — Defined in Phase 8 as `POST /api/v1/batches/{name}/revalidate` but no frontend caller. Curators cannot re-run validation after editing rules.

7. **Backend AuditEntry comments stale** — Documentation-only drift; runtime accepts Phase 11 values because the fields are plain `str`.
